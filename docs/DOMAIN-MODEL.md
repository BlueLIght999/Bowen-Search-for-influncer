# 博闻 — 领域模型

博闻的核心不是“视频榜单”，而是“帮助创作者从快速增长范例里提炼可拍摄选题方案”。

## 1. 核心域

### 1.1 Trend Discovery 热点发现

职责：按品类发现近五日快速增长的视频范例。

实体和值对象：

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| `Category` | 值对象/枚举 | 时评热点、知识科普、职场成长、商业分析、AI科技、教育观察 |
| `VideoTrend` | 实体 | 视频链接、标题、作者、发布时间、互动数据、增长分 |
| `GrowthBenchmark` | 值对象 | 同品类平均增长速度、阈值、样本数量 |
| `TrendFetchResult` | DTO | source、updatedAt、videos |

核心规则：

- 只看近五日。
- 10 万播放是 MVP 入榜硬阈值。
- 增长速度要显著高于同品类基准，默认 1.5 倍。
- Top10 结果必须稳定排序。

### 1.2 Reference Analysis 范例解析

职责：解析爆款范例为什么有效。

实体和值对象：

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| `ReferenceVideo` | 实体 | 趋势视频 + 页面详情 + 可用文案 |
| `SampleAnalysis` | 值对象 | hook、文案结构、情绪触发、场景、节奏、收藏点 |
| `CopyStructure` | 值对象 | 开头、冲突、论证、转折、行动召唤 |
| `ScenePattern` | 值对象 | 口播、屏录、访谈、实拍、混剪等 |

核心规则：

- 优先分析真实标题、简介、评论信号和字幕/文案。
- 缺失字段时允许规则兜底，但必须标记可信度。
- 分析要解释“为什么增长”，不是复述内容。

### 1.3 Strategy Recommendation 策略建议

职责：把范例解析转成可执行拍摄方案。

实体和值对象：

| 名称 | 类型 | 说明 |
| --- | --- | --- |
| `KnowledgeItem` | 实体 | 策略条目，带适用条件 |
| `DifferentiatedDirection` | 值对象 | 差异化角度、爆点策略、拍摄建议、分镜大纲 |
| `GeneratedPlan` | DTO | 面向 UI 的完整建议结果 |

核心规则：

- 每个建议必须包含差异化理由。
- 每个方向都要有可拍摄大纲。
- 输出不能只给抽象建议，必须落到镜头、标题、结构或表达方式。

## 2. 支撑域

### 2.1 Data Acquisition 数据采集

能力：

- 平台榜单抓取。
- 视频详情页抓取。
- 历史快照存储。
- 失败降级。

### 2.2 Knowledge Base 知识库

能力：

- 按品类召回策略。
- 按范例信号召回策略。
- 后续可替换为向量召回。

### 2.3 Analysis Runtime 分析运行时

能力：

- 规则分析。
- LLM 分析。
- 成本、超时和错误降级。

## 3. 通用域

- HTTP 响应封装。
- 日志和 traceId。
- 本地缓存。
- 配置读取。
- 测试 fixture。

## 4. 领域语言

统一使用以下词汇，避免同一概念多种叫法：

| 中文 | 代码建议 | 含义 |
| --- | --- | --- |
| 品类 | `Category` | 用户选择的内容赛道 |
| 热点/趋势 | `VideoTrend` | 快速增长的视频范例 |
| 同品类基准 | `GrowthBenchmark` | 同品类候选视频平均增长速度 |
| 范例解析 | `SampleAnalysis` | 对爆款文案和拍摄方式的拆解 |
| 知识条目 | `KnowledgeItem` | 可复用策略 |
| 差异化方向 | `DifferentiatedDirection` | 可执行选题建议 |
| 拍摄大纲 | `outline` | 分镜/脚本结构 |

一句话：博闻的领域模型围绕“增长范例 -> 可解释拆解 -> 可拍摄方案”组织。
