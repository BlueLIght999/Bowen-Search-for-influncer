# 「博闻」选题闭环操作系统 — 大纲PRD

> *产品定位：观点类创作者的选题闭环操作系统*
>
> *"MVP is simply for whatever the hypothesis is that we're trying to test, what is the most efficient way to get the validation we need."* — Eric Ries

> *"Use Wizard of Oz testing to validate product value propositions before investing in engineering resources."* — Crystal W

## 一、用户群体

### 1.1 核心用户：0-1万粉观点类5分钟内容创作者

| 维度 | 画像 |
|------|------|
| 身份 | 在校生/职场新人/自由职业者，想做观点类内容但缺乏选题方法论 |
| 内容类型 | 时评热点评论、知识科普、行业分析、评测测评、观点输出（访谈为V2扩展） |
| 平台 | 抖音/B站/小红书/播客，以5分钟左右中长视频或图文为主 |
| 痛点排序 | ①不知道聊什么（选题空白）> ②不知道怎么找角度（同质化严重）> ③不知道怎么组织内容（框架零散） |
| 当前解法 | 刷竞品模仿→蹭热点→凭感觉，无系统化工具 |
| 付费意愿 | 低（月入中位数1200元），但愿为"一次出圈"按次付费 |

**为什么选小白而非成熟创作者？**
- 成熟创作者已有选题直觉和方法论，工具是"锦上添花"
- 小白处于"有表达欲但无方法论"状态，工具是"雪中送炭"
- 小白基数大（72.3%创作者<10万粉），且观点类内容正在爆发期，新入局者持续涌入

**为什么从"访谈"扩展到"观点类"？**
- 选题焦虑是所有观点类创作者的共同痛点，不是访谈专属
- 时评/科普选题频率更高（每周1-3期 vs 访谈每2周1期），更容易验证闭环
- 市场规模从10-20万访谈创作者跳跃到300-500万观点类创作者
- 知识类内容播放量同比+247%，远超娱乐类+31%（QuestMobile）

### 1.2 MVP入口策略：先做"时评+科普"

| 入口类型 | 选题频率 | 用户基数 | 信号丰富度 | 付费动机 | 优先级 |
|----------|---------|---------|-----------|---------|--------|
| 时评/热点评论 | 每日-每周 | 高 | 高（热点密集） | 强（流量变现直接） | **MVP** |
| 知识科普 | 每周1-3期 | 高 | 中 | 中 | **MVP** |
| 行业分析 | 每周1-2期 | 中 | 中 | 强（专业变现） | V2 |
| 评测/测评 | 每周1-3期 | 中 | 中 | 中 | V2 |
| 访谈/对话 | 每1-2周1期 | 低 | 低 | 弱 | V2 |
| 观点输出 | 每日-每周 | 高 | 高 | 中 | V2 |

### 1.3 扩展用户（V2+）

- 1-10万粉中腰部创作者：选题pipeline管理需求
- MCN/工作室：批量管理多个账号的选题排期
- 品牌方内容团队：品牌观点类内容选题规划

## 二、市场机会

### 2.1 三重信号验证

| 信号维度 | 数据 | 来源 |
|----------|------|------|
| 观点类内容爆发 | 知识类内容播放量同比+247%，远超娱乐类+31% | QuestMobile/原创力文档 |
| 创作者规模增长 | 抖音知识类创作者120万+，10万粉+同比增长28% | 科普短视频报告/巨量算数 |
| 选题是第一痛点 | 选题占爆款成功要素54%，70%+账号流量焦虑 | 原创力文档 |

| 中长视频趋势 | 财经产业分析类中长视频占比47%，金融类中长视频互动量+54% | 抖音2026财经报告 |

### 2.2 竞品空白与格局

**15个主流访谈/内容工具全部聚焦后制作（剪辑/字幕/分发），前链路（选题/提纲/邀约）完全空白。** — PostGun调研

| 竞品 | 核心能力 | 与博闻的差异 |
|------|---------|-------------|
| **表答**（biaoda.me） | 爆款数据采集+规律分析 | 做的是"看别人怎么爆"，我们做"你怎么和别人不一样"；无闭环反馈 |
| 扣子APP | 可自建选题工作流 | 需要用户自己组装，门槛高，非开箱即用垂类工具 |
| 蝉妈妈/飞瓜 | 平台热度数据分析 | 只做数据参考，无差异化评分、无内容模板输出 |
| 剪映 | 简易口播文案生成 | 无选题热点查询能力，脚本格式单一 |
| 即梦 | 情感类脚本生成 | 无热点挖掘能力，需用户自定选题 |

**核心差异定位：**
- 表答 = 数据采集 + 爆款模仿
- 博闻 = **爆款拆解 + 差异化选题 + 知识库建议 + 闭环反馈**

### 2.3 算法适配窗口

抖音2026年算法核心变革——**收藏率>完播率**——直接利好观点类深度内容：
- 收藏率及格线3%，优质线≥5%
- 观点类内容天然触发收藏（"这段我要存下来反复看"）
- 但前提是选题切中用户"需要反复看"的需求 → 选题工具的价值被算法放大
- 权重排序：收藏率>复访率>铁粉互动>5秒留存>完播率>评论质量>点赞>转发
- 考核周期：24小时→7天长效赛马
- 搜索流量占比突破50%

## 三、核心功能

### 3.1 架构总览：上游通用 + 下游模板化

```
┌───────────────────────────────────────────────────────┐
│                上游通用层（所有内容类型共用）              │
│                                                       │
│  ①品类/热点选择 ─→ ②样本拆解 ─→ ③知识库增强 ─→ ④可行性评估 │
│  品类框架一样、拆解方法一样、差异化逻辑一样、可行性判断一样  │
└───────────────────────┬───────────────────────────────┘
                        │
             ┌──────────┼──────────┐──────────┐
             ▼          ▼          ▼          ▼
       ┌──────────┐┌──────────┐┌──────────┐┌──────────┐
       │ 时评模板  ││ 科普模板  ││ 评测模板  ││ 访谈模板  │
       │观点框架   ││知识降维   ││评测维度   ││提纲+追问  │
       │+论据素材  ││+案例库    ││+对比矩阵  ││+嘉宾推荐  │
       └──────────┘└──────────┘└──────────┘└──────────┘
                下游模板层（按内容类型差异化输出）
```

**架构优势：**
- 核心引擎（样本拆解+知识库召回+差异化评分）只建一次
- 每增加一种内容类型 = 加一个输出模板
- 新格式上线成本极低（1-2周加一个模板）

### 3.2 六阶段功能定义

#### ① 品类/热点输入（Category & Hotspot Intake）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 品类选择 | 用户先选择时评、科普、职场、商业、AI、教育等内容品类 | ✅ |
| 热点输入/选择 | 用户手动输入热点，或从预置热点池选择 | ✅ |
| 样本视频输入 | 用户粘贴公开视频链接、上传视频/截图，或粘贴标题/字幕/文案 | ✅ |
| 热搜/热榜监控 | 抖音/B站/小红书实时热榜自动采集 | ❌ V2 |
| 搜索词趋势 | 长尾搜索词、上升查询词 | ❌ V2 |
| 竞品选题库 | 同垂类账号近期选题追踪 | ❌ V2 |

**关键产品设计：MVP不做自动抓全网视频**——第一版优先让用户主动提供样本，降低平台反爬、版权和登录态风险。系统关注"把一个热点和一组参考样本转化为可拍的差异化方案"。

#### ② 样本拆解（Reference Video Analyzer）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 文案逻辑拆解 | 从标题、字幕、转写或粘贴文案中拆解开头钩子、冲突、论点推进、案例和结尾 | ✅ |
| 拍摄场景分析 | 基于视频/截图分析场景、人物出镜、字幕样式、镜头节奏、道具和画面密度 | ✅（给出建议级判断） |
| 爆点机制提取 | 识别样本的情绪触发、反常识点、信息增量、评论诱因和收藏触发点 | ✅ |
| 评论区粘贴解析 | 粘贴竞品评论区内容，提取"求展开"、争议点和二次选题信号 | ✅ |
| 平台自动抓取 | 自动下载/批量抓取平台视频、字幕和评论 | ❌ V2 |

**关键产品设计：样本拆解不是模仿，而是提取结构。** 系统要把参考视频拆成"为什么有效"，再转译成用户自己的品类、定位和资源条件能执行的方案。

#### ③ 知识库增强与差异化加工（Knowledge-Augmented Differentiation Engine）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 知识库召回 | 调取品类方法论、爆点策略、差异化策略、拍摄模板、历史案例 | ✅ |
| 蓝海选题推荐 | 基于品类、热点和样本拆解输出3个差异化选题方向 | ✅ |
| 2维差异化评分 | 角度独特性(0-100) + 竞争密度(0-100) | ✅ |
| 角度翻转建议 | 对立翻转/人群下钻/维度升降/跨域嫁接 | ✅（嵌入推荐结果） |
| 拍摄执行建议 | 给出适合用户资源条件的场景、镜头、口播节奏和素材清单 | ✅ |
| 3维评分（+收藏触发概率） | 完整版差异化评分 | ❌ V2 |

**四种差异化策略：**
1. **对立翻转**：别人做"如何成功"→ 你做"那些失败的人后来怎样了"
2. **人群下钻**：别人做泛人群 → 你做"30岁转行的人""小镇青年视角"
3. **维度升降**：别人讲方法论 → 你讲故事；别人讲故事 → 你讲数据
4. **跨域嫁接**：AI话题 × 教育视角、职场话题 × 心理学框架

**关键洞察：** 选题占爆款成功要素54%，但AI选题爆款率<0.2%。不是缺选题，是缺差异化选题。差异化评分引擎是核心壁垒。

#### ④ 可行性评估（Feasibility Gate）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 创作者匹配度快判 | "这个选题和你的定位一致吗？" 简单判断 | ✅（手动勾选） |
| 风险扫描 | 敏感词/争议性预警 | ❌ V2 |
| 嘉宾可达性（访谈模板专属） | 你能找到人聊这个吗？ | ❌ V2 |

#### ⑤ 选题排期（Topic Pipeline Manager）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 选题卡片 | 确认选题后生成卡片，含选题名+差异化评分+预期内容类型 | ✅ |
| Pipeline看板 | 候选→待确认→已排期→制作中→已发布 | ❌ V2 |
| 新鲜度衰减 | 热点类7天保质期提醒 | ❌ V2 |

#### ⑥ 发布复盘（Performance Attribution）

| 功能点 | 描述 | MVP包含 |
|--------|------|---------|
| 手动回填收藏率 | 发布后手动输入收藏率，标记正/负样本 | ✅ |
| 评论区选题提取 | 自动扫描评论区→回流到① | ❌ V2 |
| 衍生选题生成 | 从一期内容拆出2-3个子话题 | ❌ V2 |

### 3.3 下游模板层：内容类型差异化输出

#### MVP模板：时评/科普观点框架

## 四、关键闭环

### 4.1 MVP闭环：最小可运转循环

```
选择品类 → 输入/选择热点 → 提交样本视频/文案 → 拆解文案逻辑+拍摄场景
    → 知识库增强建议 → 3个差异化方案+2维评分 → 确认选题 → 生成可拍摄内容模板
       ↑                                                                      │
       └──── 发布后手动回填收藏率/播放表现 ←── 评分校准 ←────────────────────────┘
```

> *Jason Fried: "Set a hard maximum of six weeks for any feature development. Treat time as a budget."*
>
> 6周预算：前4周打通主链路，后2周接入回填闭环。

### 4.2 三条回流管道（V2渐进开放）

| 管道 | 闭环路径 | 预期效果 | 开放时间 |
|------|---------|---------|---------|
| A. 评论区→选题信号 | 发布→评论"求展开"→自动提取→候选池 | 30%+新选题来自回流 | V2 |
| B. 数据归因→评分校准 | 选题评分→发布→收藏率→正/负样本→模型优化 | 评分准确率周提升5%+ | V2渐进 |
| C. 内容网络扩散 | 一期内容→评论区衍生话题→新选题→新内容 | 每期内容拓展1.5个新方向 | V3 |

### 4.3 完整闭环架构

```
┌───────────────────────────────────────────────────────────────┐
│                                                               │
│  ①品类/热点 ─→ ②样本拆解 ─→ ③知识库增强/差异化 ─→ ④可行性评估 ─→ ⑤发布复盘 │
│       ↑                                                   │        │
│       │        ┌──────── 三条回流管道 ────────┐            │        │
│       │←── A.评论区二次选题 ←────────────────│            │        │
│       │←── B.数据归因反哺 ←──────────────────│            │        │
│       │←── C.内容网络扩散 ←──────────────────┘            │        │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 4.4 闭环健康度指标

| 指标 | 定义 | MVP目标 |
|------|------|---------|
| 选题到发布周期 | 品类/热点输入→发布上线平均天数 | <14天 |
| 回流选题占比 | 来自评论区/复盘的新选题比例 | 手动回填率>50%即算通过 |
| 评分校准度 | 差异化评分与实际收藏率的相关性 | 积累50个样本后检验 |
| Pipeline储备量 | 候选选题数量 | >5个 |

## 五、用户验证

### 5.1 核心假设（需验证）

| # | 假设 | 验证方法 | 成功标准 |
|---|------|---------|---------|
| H1 | 小白创作者的最大痛点是"不知道聊什么/找不到差异化角度" | 用户访谈(n=10) | 7/10确认选题是第一痛点 |
| H2 | 用户愿意为"一次出圈选题"按次付费 | 付费意愿测试 | 5/10表示愿意付9.9-29.9元/次 |
| H3 | 样本拆解+知识库建议比单纯AI选题更能让用户产生"我知道怎么拍了"的确定感 | WoZ测试后访谈 | 4/5表示方案可直接进入拍摄准备 |
| H4 | 差异化评分和实际表现正相关 | 回填数据验证 | 50个样本后评分与收藏率r>0.3 |
| H5 | 观点类创作者（非访谈）也有强烈的选题差异化需求 | 扩展用户访谈 | 6/10非访谈创作者确认选题差异化是痛点 |

### 5.2 验证优先级

> *Crystal W: "Use Wizard of Oz testing to validate product value propositions before investing in engineering resources."*

**先做Wizard of Oz验证，再写代码：**
1. 用微信1v1手动模拟"品类/热点/样本输入→样本拆解→知识库建议→差异化方案"→ 观察用户是否真的愿意按方案准备内容
2. 用ChatGPT+手动编排模拟完整流程 → 确认用户是否能从方案进入拍摄执行
3. 积累10个付费意愿样本 → 再决定是否投入开发
4. 特别验证非访谈类创作者（科普/时评）的选题需求是否同样强烈

## 六、商业判断与假设

### 6.1 商业模式：按次付费 + 订阅
